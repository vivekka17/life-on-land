Splash Page

Technical Description

The Splash Page was implemented using a full-screen background image with a semi-transparent overlay. An external CSS file (splash.css) applies a flexbox layout to center the content vertically and horizontally. A meta refresh tag redirects users after 4 seconds, and a CSS spinner animation provides visual feedback during the transition.

Accessibility

Accessibility was ensured by using semantic HTML elements such as <header>, <main>, and <footer>, providing alternative text for images, and maintaining sufficient color contrast between text and background.

Volunteer Page

Technical Description

The Volunteer Page combines HTML, CSS, and JavaScript to create an interactive experience. A continuous marquee effect (with duplicated content for a seamless loop and a hover pause) is applied to the program cards. Consistent card sizing, CSS transitions, and hover effects ensure a responsive and engaging user interface.

Accessibility

Descriptive alt text for images, clear focus states for interactive elements, and semantic tags were used to improve accessibility. The marquee pauses on hover, enabling users to interact with the cards without distraction.

Content Page

Technical Description

The Content Page (Forest Conservation) uses a structured layout with internal navigation. Headings, paragraphs, and images are styled consistently via CSS. Responsive design (including media queries) ensures the layout adapts to various screen sizes. Semantic HTML provides logical structure and improves accessibility.

Accessibility

The Content Page meets accessibility standards by employing a clear heading hierarchy, descriptive alt text for images, and proper color contrast. Internal navigation links help users easily jump between sections on the page.

Challenges and Lessons Learned

A key challenge was maintaining a unified look across pages while incorporating unique features (like the marquee on the Volunteer Page). Debugging the seamless animation and ensuring hover-to-pause required iterative testing. Through this process, I learned the importance of duplicating content for a continuous loop and balancing interactive effects with overall usability.

Compliance

All pages have been validated using W3C tools and comply with HTML and CSS standards. Accessibility guidelines—such as proper color contrast, semantic markup, and descriptive alt text—were followed to ensure usability for a wide range of users.
